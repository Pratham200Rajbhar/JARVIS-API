📚 JARVIS AI Documentation

Complete API reference and usage guide for the advanced AI system

🎯 Overview

JARVIS AI is a sophisticated multi-provider AI system that intelligently routes requests to the best available AI service. It features automatic fallback mechanisms, priority-based routing, and comprehensive error handling.

🌟 Key Features

  • Multi-Provider Support: Groq, OpenAI, OpenRouter, and Gemini integration
  • Intelligent Fallback: Automatic provider switching on failures
  • Priority Routing: Configurable provider priority system
  • Real-time Monitoring: Comprehensive logging and statistics
  • RESTful API: Clean, consistent API design

🚀 Quick Start

Get started with JARVIS AI in minutes. Here's how to make your first API call:

# Start the server
npm start

# Make a test request
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello, JARVIS!"}'

🔧 Installation

Clone the repository and install dependencies:

git clone https://github.com/your-repo/jarvis-ai.git
cd jarvis-ai
npm install
npm start

🚨 Error Handling

Understanding JARVIS AI's error handling and response codes.

📋 Error Response Format

All errors follow a consistent format:

{
  "success": false,
  "error": {
    "message": "Human readable error message",
    "code": "ERROR_CODE",
    "timestamp": "2024-01-15T10:30:00Z",
    "details": {
      // Additional error context
    }
  }
}

🔢 Common Error Codes

  • INVALID_REQUEST - Malformed request body
  • NO_PROVIDERS_AVAILABLE - All providers are down
  • RATE_LIMIT_EXCEEDED - Too many requests
  • PROVIDER_ERROR - AI provider returned an error
  • INTERNAL_ERROR - Server internal error

⏱️ Rate Limiting

Information about API rate limits and best practices.

📊 Current Limits

  • Chat Endpoint: 100 requests per minute
  • Admin Endpoint: 60 requests per minute
  • Status Endpoint: 200 requests per minute

🎯 Best Practices

  • Implement exponential backoff for retries
  • Cache responses when appropriate
  • Use batch requests for multiple operations
  • Monitor your usage with the admin endpoint

📖 Examples

Practical examples of using JARVIS AI in different scenarios.

🐍 Python Example

import requests
import json

def chat_with_jarvis(message):
    url = "http://localhost:3000/api/chat"
    payload = {"message": message}
    
    response = requests.post(url, json=payload)
    
    if response.status_code == 200:
        data = response.json()
        if data["success"]:
            return data["data"]["response"]
        else:
            return f"Error: {data['error']['message']}"
    else:
        return f"HTTP Error: {response.status_code}"

# Usage
response = chat_with_jarvis("What's the weather like?")
print(response)

📜 JavaScript Example

async function chatWithJarvis(message) {
    try {
        const response = await fetch('/api/chat', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({ message })
        });
        
        const data = await response.json();
        
        if (data.success) {
            return data.data.response;
        } else {
            throw new Error(data.error.message);
        }
    } catch (error) {
        console.error('Error:', error);
        return null;
    }
}

// Usage
chatWithJarvis('Hello, JARVIS!')
    .then(response => console.log(response))
    .catch(error => console.error(error));

    Error Response Format

    JSON
    {

  "error": {
    "message": "All AI providers failed",
    "details": "No providers available or all providers returned errors",
    "code": "PROVIDERS_UNAVAILABLE",
    "attempts": [
      {
        "provider": "groq",
        "error": "API key invalid"
      },
      {
        "provider": "github",
        "error": "Rate limit exceeded"
      }
    ]
  },
  "timestamp": "2025-07-18T10:30:00.000Z"
}

    Common Error Codes

    Code Description HTTP Status
    INVALID_REQUEST Invalid request format or missing parameters 400
    PROVIDERS_UNAVAILABLE All AI providers failed or are unavailable 503
    RATE_LIMIT_EXCEEDED Too many requests 429
    INTERNAL_ERROR Internal server error 500

    ⚠️ Important

    When all providers fail, the system returns a 503 Service Unavailable status. The fallback system ensures maximum uptime by trying all available providers before failing.
